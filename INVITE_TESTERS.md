# テストユーザー招待手順

## 🎯 結論：目的別の3つのモード

| モード | URL例 | 何ができる |
|--------|------|-----------|
| **A. UI評価のみ**（最も簡単） | `https://kodawarimax.github.io/ai-creator/` | 画面遷移・デザイン評価・💬フィードバック投稿 |
| **B. フル機能テスト** | `https://kodawarimax.github.io/ai-creator/?token=ABC123XYZ` | A + AI生成・PDF処理・全API機能 |
| **C. ローカル開発** | `http://localhost:5174` | 全機能 + バックエンド改修 |

---

## モードA: UI評価のみ（招待コード不要）

**そのままURLを共有するだけ:**

```
https://kodawarimax.github.io/ai-creator/
```

- 画面右上に **🟡 UI評価モード（API未接続）** バッジが表示
- AI生成ボタンを押すとエラーになるが、UI/画面遷移/💬フィードバックは全て利用可能
- デザイナーレビューには十分

→ デザイナー10人に同時に配ってOK、何の管理も不要

---

## モードB: フル機能テスト（トークン配布）

### Step 1: トークンを生成（管理者作業、1回のみ）

```bash
openssl rand -hex 24
# 例: 9f3a2b7c8d1e6f4a5b9c2d7e3f8a1b4c9d6e2f7a8b3c5d1e
```

### Step 2: VPSとGitHub Secretsに登録

`SECURITY.md` 参照（既に設定済みなら省略）

### Step 3: テスター招待URL作成

```
https://kodawarimax.github.io/ai-creator/?token=<手順1のトークン>
```

このURLをテスターに送信。

### Step 4: テスターが体験すること

1. URLをクリック
2. ブラウザが自動でトークンを `localStorage` に保存
3. URLからトークンが消えてクリーンになる（共有しても安全）
4. 画面右上に **🟢 API接続OK** バッジが表示
5. 全機能が使える

### Step 5: 終了後（任意）

トークンを無効化したい場合:
- 新トークンを生成して再デプロイ → 旧トークンは即座に失効

---

## ステータス表示の見方

| バッジ | 意味 | 対処 |
|--------|------|------|
| 🟢 API接続OK | バックエンド到達 + 認証成功 | OK |
| 🟡 UI評価モード | トークン未設定 | UIだけ評価可、AI機能は使えない |
| 🔴 トークン無効 | トークンが古いか間違い | 新URLを管理者に依頼 |
| 🔴 バックエンド到達不可 | VPSダウン or ネットワーク問題 | しばらく待って再読込 |

## トークン配布のコツ

### 信頼できる相手に配る場合
- DM/メール/Slackで `?token=...` 付きURLを送信
- 一つのトークンを複数人で共用OK（レートリミットはIPごと）

### 不特定多数に配る場合
- 推奨しない。トークンがバレるとAPIキー消費される
- 必要なら期間限定トークン → デモ後に新トークン生成 → ローテーション

### テスター個別管理したい場合
- 現状の仕組みでは無理（簡易認証のため）
- 必要なら Supabase JWT認証への移行を検討

## トラブルシューティング

### Q. テスターが「🔴 トークン無効」と言う
- A. ブラウザの devtools → Application → Local Storage → `aic_api_token_v1` を確認
- 値が空 → URLパラメータが正しく付いていない
- 値が古い → 新トークンURLを再送

### Q. 一度入れたトークンを別アカウント用に上書きしたい
- A. 新URL `?token=新値` を開けば自動で上書き

### Q. トークンを完全に消したい
- A. ブラウザコンソールで `localStorage.removeItem('aic_api_token_v1')` または devtools から削除
