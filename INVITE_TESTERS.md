## 将来的に認証を強制したくなったら

`SECURITY.md` の手順で:

```bash
# VPS上で
sudo systemctl edit ai-creator.service
# [Service]
# Environment="API_ACCESS_TOKEN=creator2026"
sudo systemctl daemon-reload
sudo systemctl restart ai-creator
```

これだけ。以降は `creator2026` 以外のパスワードは拒否されます。

### 終了時
1. テスター集計 → フィードバック取込
2. パスワードをローテーションしたい場合:
   - `INVITE_TESTERS.md` 編集で新パスワード記載
   - VPS `API_ACCESS_TOKEN` を新値に更新 → `systemctl restart`

### 本番運用に切り替える時
- 共有パスワード廃止 → Supabase JWT 認証導入（開発ロードマップ参照）

## トラブルシューティング

### Q. テスターが「画面が真っ暗／何も出ない」と言う
- ブラウザキャッシュの可能性。ハードリロード（Cmd+Shift+R / Ctrl+F5）を依頼
- それでもダメなら devtools → Console のエラーログをスクショで送ってもらう

### Q. 「💬フィードバックの一覧が消えた」
- localStorage がクリアされた可能性（シークレットモード等）
- テスターのブラウザ依存なので復元不可

### Q. パスワードを変えたい
- `INVITE_TESTERS.md` のこのファイルで管理しているので、`creator2026` を新値に置換
- VPS の `API_ACCESS_TOKEN` を同じ新値に更新
- 全テスターに新パスワードを再配布
