# テストユーザー招待手順

## 🎯 結論：パスワード方式（最もシンプル）

URL は **常に同じ**:
```
https://kodawarimax.github.io/ai-creator/
```

テスターはサイト右上の **🟡 UI評価モード** ボタンをクリック → パスワード入力 → **🟢 AI機能 ON** に切り替わる。

---

## 2つのテストモード

| モード | 必要なもの | 何ができる |
|--------|-----------|-----------|
| **A. UI評価モード**（デフォルト） | URL のみ | 画面遷移・デザイン評価・💬フィードバック投稿 |
| **B. AI機能 ON** | パスワード | A + AI生成・PDF処理・全API機能 |

---

## モードA: UI評価のみ（招待コード不要）

URL を共有するだけ:
```
https://kodawarimax.github.io/ai-creator/
```

- 画面右上に **🟡 UI評価モード（クリックで接続）** バッジ
- AI生成は使えないが、画面遷移・デザイン評価・💬フィードバック投稿は全部OK
- **デザイナー10人に同時に配ってOK、何の管理も不要**

## モードB: AI機能を有効化（パスワード配布）

### Step 1: パスワードを決める（管理者）

```bash
openssl rand -base64 12
# 例: K7gPq2mWxYz9Vn4R
```

または覚えやすい文字列でもOK（例: `creator-test-2026-spring`）

### Step 2: VPSとGitHub Secretsに登録

両方に**同じパスワード**を設定（`SECURITY.md` 参照）

| 場所 | 環境変数名 | 値 |
|------|-----------|---|
| VPS (srv1334941) | `API_ACCESS_TOKEN` | パスワード |
| GitHub Secrets | `VITE_API_TOKEN` | （任意・空でも可） |

> GitHub Secrets は省略可能。フロント側はパスワードを毎回ユーザに入力させる方式に変更したため、ビルド時に埋め込まなくても動きます。

### Step 3: テスターに渡すもの

DM/メールで以下2点を送るだけ:

```
URL: https://kodawarimax.github.io/ai-creator/
パスワード: K7gPq2mWxYz9Vn4R
```

### Step 4: テスターの操作

1. URL を開く
2. 右上の **🟡 UI評価モード** バッジをクリック
3. パスワード入力モーダルが開く → パスワード入力 → 「接続」
4. **🟢 AI機能 ON** に切り替わる（次回以降は自動）
5. AI機能が全部使える

### Step 5: ログアウト

ユーザー側: バッジクリック → 「🚪 ログアウト」  
管理者側: パスワード変更 → VPS再起動 → 全テスター失効

---

## ステータスバッジの見方

| バッジ | 意味 | 対処 |
|--------|------|------|
| 🟢 AI機能 ON | バックエンド到達 + 認証成功 | OK、何もしない |
| 🟡 UI評価モード | パスワード未入力 | UI評価のみ可能、AI機能はクリックで接続 |
| 🔴 パスワード再入力が必要 | パスワードが変更された | クリックして再入力 |
| 🔴 バックエンド到達不可 | VPSダウン | 時間を置いて再読込 |

---

## 配布のコツ

### 信頼できる相手に配る場合
- 1つのパスワードを複数人で共用OK（IPごとレートリミット 60req/min, 300req/h）
- DM/メール/Slackで気軽に共有

### テスト終了後
- VPSの `API_ACCESS_TOKEN` を新値に変更 → `systemctl restart ai-creator`
- 既存テスターは自動失効 → 必要なら新パスワード送付

### 不特定多数に配る場合
- 推奨しない。パスワードがバレるとAPI消費される
- やるなら短期限定 → 終了後ローテーション

---

## トラブルシューティング

### Q. 「パスワードが違います」と出る
- 管理者にパスワード再確認を依頼。前後の空白に注意

### Q. 「サーバーに接続できません」と出る
- VPSがダウンしている可能性。管理者に連絡

### Q. パスワードを完全に忘れさせたい
- ユーザーがバッジクリック → 「🚪 ログアウト」
- またはブラウザ devtools で `localStorage.removeItem('aic_api_token_v1')`

### Q. 「あとで」を選んだが後から接続したくなった
- 右上の 🟡 バッジを再度クリック → パスワード入力
